# Prevention Defenses

**Type:** method

Security mechanisms that strengthen oracle function approximations through prompt engineering, model training, policy specification, and privilege separation to improve agent security.

---

Prevention defenses are security mechanisms that aim to strengthen the approximations of oracle functions before a potential violation occurs, reducing the likelihood that an agent will enter an insecure state in the first place. Techniques in this category include prompt engineering to clarify task scope and source boundaries, model fine-tuning and safety alignment training to internalize security-relevant behaviors, formal policy specification to define allowed actions and objectives, and privilege separation to limit what capabilities the agent can invoke in a given context. Within the contextual security framework, prevention defenses are understood as mechanisms that improve the agent's ability to accurately evaluate task alignment, action alignment, source authorization, and data isolation during execution. Their effectiveness is bounded by the quality of their approximation of the corresponding oracle functions and the completeness of the policy specifications they enforce.
