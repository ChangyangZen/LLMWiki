# Agent Contracts

**Type:** model

A formal framework defining agent execution as a seven-tuple C=(I,O,S,R,T,Φ,Ψ) that unifies input/output specifications, resource constraints, temporal boundaries, and success criteria into a coherent governance mechanism.

---

Agent Contracts is a formal framework introduced to govern autonomous AI agents through explicit resource and temporal constraints, defined as a seven-tuple C=(I,O,S,R,T,Φ,Ψ) encompassing input specifications, output specifications, state, resource constraints, temporal boundaries, success criteria, and termination conditions. The framework extends the Contract Net Protocol metaphor from task allocation to resource-bounded execution, providing a coherent governance mechanism for both single-agent and multi-agent systems. Empirical validation across four experiments demonstrated significant practical benefits, including 90% token reduction in iterative workflows, 525× lower variance in code review pipelines, and zero conservation violations in multi-agent delegation hierarchies. By unifying specification, resource governance, and lifecycle management into a single formal structure, Agent Contracts operationalize principles of bounded rationality and satisficing for production AI deployments.
